## **🚗 Automotive（汽车行业）名词解释**

- **OEM（Original Equipment Manufacturer）**

  原始设备制造商，负责整车设计、组装和销售的厂商。例如：Toyota（丰田）、Volkswagen（大众）、BMW（宝马）。
  

- **AUTOSAR（Automotive Open System ARchitecture）**

  一种开放、标准化的汽车软件架构，旨在提升车载软件的**可复用性、模块化与标准化**。

  AUTOSAR 分为两个主要平台：
  
  
  - **Classic Platform（CP）**：适用于传统嵌入式 ECU，使用 C 语言开发，资源受限，适合周期性控制任务。
  - **Adaptive Platform（AP）**：适用于高性能计算场景（如自动驾驶、车载 AI），支持 C++14、POSIX、多线程及服务导向架构（SOA）。
  
  
  
- **ASPICE（Automotive SPICE）**

  **汽车软件过程改进与能力评估模型**，是一种用于评估汽车嵌入式软件开发过程质量的国际标准，广泛用于 Tier1/Tier2 与 OEM 合作中的**流程认证**。
  

- **AAOS（Android Automotive OS）**

  基于 AOSP 的 Android 车载操作系统，是 Google 推出的原生车机系统。运行于车载硬件之上，支持运行应用、渲染系统 UI
  

## 通信方式

```
                       [ HMI App ]
                            ↓
                [     AIDL/Service     ]
                      ↓          ↓
         ┌────────────────┐   ┌──────────────┐
         │ JNI (SocketCAN)│   │ JNI (vsomeip)│
         └──────┬─────────┘   └───────┬──────┘
                ↓                     ↓
         [ CAN 驱动 / 硬件]    [ Ethernet 通信 ]
                ↓                     ↓
            [ECU-1]            [ECU-2 / 网关]
```
