## ASPICE（Automotive SPICE）软件工程核心过程



### SWE.1 - Software Requirements Analysis

- **目的**：分析系统需求，提取出与软件相关的需求。

- **关键活动**：

  - 从系统需求中派生出软件需求（SRD）。
  - 明确接口、功能、安全性、性能等需求。
  - 与系统团队协作，澄清不明确的部分。

  

- **输出物**：

  - 软件需求规格说明（SRS）

  - 软件接口说明（SIS）

    

### SWE.2 - Software Architectural Design（软件架构设计）

- **目的**：定义系统级软件的高层结构概览，包括主要模**模块，类，接口 与 之间关系**
  将需求分配到架构元素中，并评估设计合理性

- **关键活动**：

  - 设计模块划分、任务调度、通信机制 
  - 定义硬件抽象、操作系统交互、安全隔离等架构。
  - 考虑可扩展性、性能和安全性。

  

- **输出物**：

  - 软件架构设计文档（SAD）
  - 软件组件与接口定义



### SWE.3 - Software Detailed Design and Unit Construction（软件详细设计与单元构建）

- **目的**：详细设计每个软件单元并实现代码。

- **关键活动**：

  - 编写每个模块的内部逻辑。
  - 开发源代码，符合编码规范。
  - 可选：设计模型（如Simulink）转代码。

  

- **输出物**：

  - 单元设计文档（SDD）
  - 源代码

  

### SWE.4 - Software Unit Verification（软件单元验证），可在Gitlab CI/CD集成

- **目的**：验证每个单元是否按设计正确实现

- **工具:**
  
  - 代码静态扫描 SonarQube
  
   - 自动化测试相关
     

| **测试类型** | **覆盖层** | **特点**                                     | 报告生成                    | 常用工具                    |
| ------------ | ---------- | -------------------------------------------- | --------------------------- | --------------------------- |
| **单元测试** | 方法级     | 快速执行，不依赖 UI 或设备，适合业务逻辑验证 | ./gradlew testDebugUnitTest | JUnit, Mockito, Robolectric |




- **输出物**：
  	- 静态分析结果
  	- 单元测试报告
  	- 代码覆盖率报告



### SWE.5 - Software Integration and Verification（软件集成与验证）

| **测试类型**    | **覆盖层** | **特点**                                             | 报告生成                            | 常用工具                           |
| --------------- | ---------- | ---------------------------------------------------- | ----------------------------------- | ---------------------------------- |
| **集成测试**    | 组件级     | 测试多个模块之间的交互，例如 ViewModel 和 Repository | 可嵌入单元测试或 UI 测试中组合测试  | JUnit, Espresso (部分), MockK      |
| **UI** **测试** | 应用级     | 模拟用户行为（如点击、输入），耗时高，需设备或模拟器 | ./gradlew connectedDebugAndroidTest | Espresso, UIAutomator, Robolectric |

- **目的**：将多个软件单元集成，并验证整体功能。
- **关键活动**：(  集成/UI 测试效率过慢 === > 周期执行/固定分支执行)
  - 集成模块，构建完整软件。
  - 执行集成测试，确保各模块协作正常。
  - 验证软件是否满足需求。
  



- **输出物**：

  - 集成测试用例和报告
  - 问题跟踪记录
  - 可执行软件包
